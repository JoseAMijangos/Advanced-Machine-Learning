  2. Neural net for credit card default problem

You job is to modify the code in section ‘Training a DNN Using Plain TensorFlow’ in Chap. 10 of our text to build a neural net  for the Credit card default data.  Do not use TF.Learn.  Use the same data set we used for Project 1.  You can use the code in the author's notebook for Chapter 10 instead of typing in the code from the textbook:

https://github.com/ageron/handson-ml/blob/master/10_introduction_to_artificial_neural_networks.ipynb

See the code near the end of the notebook, just under 'Using dense() instead of neuron_layer()'.

You will submit your work as an .ipynb file and the derived .html file.  

If you're not sure where to start, here are some ideas:

  • Start by making sure you can run the code from Chapter 10 without modification.
  • You will need to add code to read in the credit card data and do any preprocessing you want.  You can use sklearn functions.
  • You will need to split the data to get X_train, X_test, y_train, y_test from your preprocessed data,  Again, you can use sklearn if you want
  • You will need to adjust the number of inputs and the number of outputs.  
  • You will need to provide a function to fetch a mini_batch (we have seen such functions in lab)
  • You will need to think about the loss function you want to use.
  • Use mini-batch gradient descent, as in the code above.
  • You may want to adjust learning rate.

Hints:

  • One of the harder things to figure out in this problem concerns the number of outputs, and whether you will use tf.nn.sparse_softmax_cross_entropy_with_logits, or something else.  My recommendation is to start by trying to keep your code as similar to the code for MNIST as possible.
  • If your net blows up right away, and you see NaN values, you may need to adjust your learning rate.

Once you get your neural net working, tune it using the ideas in Chap. 10 of our text.  You can play with lots of things, like the optimizer you use, how initialization is performed, choice of activation function, and the number and size of layers in your net.
