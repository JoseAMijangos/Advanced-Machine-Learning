Voting.  At the beginning of Chapter 3, the author shows how the load the MNIST dataset of small images of handwritten digits.  Using this dataset, build a voting classifier.   Use the Scikit-Learn VotingClassifier.  Use three classification algorithms as the base classifiers: Logistic Regression, Support Vector Machines, and Random Forests.  

Update: The MNIST dataset mentioned above is large and might be slow to process.  It is acceptable to use the smaller Scikit-Learn digits dataset.  See http://scikit-learn.org/stable/auto_examples/datasets/plot_digits_last_image.html for details.

This is a multiclass problem (label is a value from 0-10), but note that Scikit-Learn will automatically use one-versus-all or one-versus-one to create a multiclass classifier if your classifier does not natively support it.  (See p. 94 in our text.)

Use 70% of the MNIST data as training data, and the rest as test data.  Compare the performance of your voting classifier with the performance of each of the base classifiers.

Create an jupyter notebook for your work.  See comments in the submission section about your notebook.

  1. Stacking.  Build your own a simple stacking regressor as a Scikit-Learn predictor class named ‘StackingRegressor’.  It should support at least fit(), predict(), and score() methods.  The constructor should have an ‘predictors’ argument that is a list of predictors.  The fit() method should have X and y arguments.  The predict() method should have an X argument.  The score() method should have X and y arguments.  You can use whatever base regressors you like (but of course the base regressors should all be different kinds of regressors).  Here is an example of how your regressor could be used:

regr = StackingRegressor([regr1, regr2, regr3])

regr.fit(X_train, y_train)

y_predict = regr.predict(X_test)

regr.score(X_test, y_test)

Create a small iPython notebook that shows the code for your new class, and shows the class being used.  You can pick the base predictors you want -- obviously they must all be regressors.  Compare the performance of your regressor against linear regression.  For data, use the built-in Scikit-Learn dataset on diabetes.
