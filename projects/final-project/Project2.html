Project 2

In this project you will develop a character-level text model using RNNs in TensorFlow.

Part 1. Your team will develop TensorFlow code that matches the text model of Andrej Karpathy that is written in Python without TensorFlow. We have studied this code in class. It can be found at: https://gist.github.com/karpathy/d4dee566867f8291f086

What does "matches" mean? It means this:

  • the sequence length (number of steps) are the same as Karpathy
  • the number of neurons per cell are the same as Karpathy
  • upper and lower case characters, and punctuation, and included in the vocabulary
  • loss, and some sample output, are reported during training

Your TensorFlow code must be structured similarly to the code in our text. The goal of this project is not to explore software design with TensorFlow. If you have lots of time, please use it to develop a RNN that produces great text (this requirement added 12/2018).

Also, you should use the Alice in Wonderland text as we did in a lab.

On the other hand, you code must be different from Karpathy's in the following ways:

  • you will not implement backprop by hand -- you must use a TensorFlow optimizer, such as the AdamOptimizer
  • you must use tf.nn.rnn_cell.BasicRNNCell and tf.nn.dynamic_rnn to build your RNN

In some areas, I give you the choice of how closely you want to stick to Karpathy's code:

  • batch size
  • how batches are selected from the input data

Part 2. After you get part 1 working, you should experiment and create a second RNN. For example, you could try different batch sizes, different sequence lengths, different number of neurons per cell, dropout, etc. Also, you may want to use an LTSM or GRU cell instead of a basic RNN cell. (See our text for how to do this.) You can use input text data of your choice. Pick something interesting, but remember you need quite a bit of data to train an RNN.

Report. Your submitted Jupyter notebook should contain the following:

  1. A title, intro, and conclusions
  2. Your base TensorFlow code (matching Karpathy), with markup code as necessary. For graph execution, you should include one cell that shows training with occasional loss and sample output reported, stopped after about a page of output with a keyboard interrupt. Another cell should a longer run with no occasional reports, but with sample output for the "fully-trained" RNN.
  3. Your extended TensorFlow code (incorporating your experiments), with markup code to note that changes from the base code. Include at least two cells related to graph execution, as mentioned above.
  4. A commentary section before your code that lists the technical issues you faced, and how you solved them. I expect you will have to face and solve at least 10 issues on the way to getting your final code. Make notes as you work on the project so you don't have to remember what the issues were later. Be concise: this section should be about 1-2 pages in length.

Notes and Hints.

  • You will surely want to do some googling, especially to understand how parts of the TF API work. Please do not refer to sources that provide a solution to the project. This will destroy the learning value of the project for you. Also, if I see that submitted code is based on web sources, I will penalize the code severely.
  • Your code must use the plain TensorFlow API, not Keras or any other API built on top of TensorFlow.
  • You will probably want to define a fetch_batch() function like we have in the past. What is the form of data that you want in a batch?
  • There are three forms of character data that you will to deal with: the original character data, characters encoded as integers, and one-hot encoded characters. An issue you'll have to face is how much data processing gets done before the data is fed to placeholders, versus how much processing gets done in the RNN itself.
  • Here is a quote from our text about dynamic_rnn() that is very important to remember:zz

Conveniently, dynamic_rnn() accepts a single tensor for all inputs at every time step (shape [None, n_steps, n_inputs]) and it outputs a single tensor for all outputs at every time step (shape [None, n_steps, n_neurons]).
